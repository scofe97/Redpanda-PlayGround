# Git 워크플로우 규칙

## PRE-COMMIT GATE (커밋 전 필수)

이 게이트를 통과하지 않고 커밋하면 안 됨:

```
□ 1. git pull 실행
□ 2. 변경사항 분석 → 커밋 분할 계획 테이블 작성
□ 3. 분할 계획 사용자에게 제시
□ 4. 사용자 승인 받음
□ 5. 승인 후 커밋 실행
```

분할 계획 형식:
| # | 주제 | 파일 | 타입 |
|---|------|------|------|
| 1 | ... | ... | feat/fix/refactor |

> 개인 프로젝트이므로 Jira 티켓번호 불필요.

## 커밋 메시지 형식

```
타입: 설명
```

### 타입 종류
| 타입 | 설명 |
|------|------|
| `feat` | 새로운 기능 |
| `fix` | 버그 수정 |
| `refactor` | 리팩토링 (기능 변경 없음) |
| `docs` | 문서 변경 |
| `test` | 테스트 추가/수정 |
| `chore` | 빌드, 설정 변경 |
| `style` | 코드 포맷팅 |

## 안전 규칙

### 금지 사항
- `git push --force` to main/master
- `git commit --amend` 후 force push
- 시크릿 포함 커밋
- **Co-Authored-By 금지** -- 커밋 메시지에 Co-Authored-By 추가하지 않음

### 커밋 전 확인
- [ ] 빌드 통과
- [ ] 테스트 통과
- [ ] 민감 정보 없음
