왜 트랜잭션 분리인가
TX-1 조회/락: PENDING → PROCESSING → NO-TX Kafka 발행 (네트워크 IO) → TX-2 성공/실패 마킹 (SENT, RETRY, DEAD)
Kafka send를 DB 트랜잭션 안에 두면 브로커 지연/타임아웃 동안 커넥션 점유가 길어져 풀 고갈 위험이 커진다. 그래서 조회/상태변경은 짧은 트랜잭션으로 끝내고, 발행은 트랜잭션 밖에서 실행한 뒤 결과만 다시 DB에 반영한다.
